1. UAT OVERVIEW

Purpose of UAT

User Acceptance Testing validates that the PNG Green Fees System meets business requirements and is ready for production deployment.

üéØ UAT Goals:
  • Verify all features work as expected
  • Ensure system meets user requirements
  • Identify any usability issues

Prerequisites

8. TEST SCENARIO 6: CASH RECONCILIATION

Test Steps

  1. Navigate to Cash Reconciliation ‚Üí Start New
  2. Enter opening float: PGK 100
  3. System shows expected cash (based on transactions)
  4. Count cash denominations:
    • 100 Kina: 2 notes
    • 50 Kina: 4 notes
    • 20 Kina: 5 notes
    • 10 Kina: 2 notes
  5. System calculates total actual cash
  6. Review variance (if any)
  7. Add notes explaining variance
  8. Click Complete Reconciliation
  9. Submit for approval

Expected Results

‚úÖ Success Criteria:
  • Opening float recorded
  • Expected cash calculated from transactions
  • Actual cash totaled correctly
  • Variance calculated (expected - actual)

  • Status shown as "Pending Approval"

13. ISSUE REPORTING

How to Report Issues

If you encounter any problems during UAT, please document them using this template:

‚ö†Ô∏è Issue Report Template:
Field Details
Issue ID Sequential number (e.g., UAT-001)
Date/Time When issue occurred
Test Scenario Which scenario you were testing
Description What went wrong (be specific)
Steps to Reproduce How to replicate the issue
Expected Result What should have happened
Actual Result What actually happened
Severity Critical / High / Medium / Low
Screenshot Attach if applicable

Severity Levels

Level Description Example
Critical System unusable, data loss Cannot login, payments fail
High Major feature broken Reports don't generate
Medium Feature partially works Search is slow
Low Minor issue, cosmetic Typo in label
